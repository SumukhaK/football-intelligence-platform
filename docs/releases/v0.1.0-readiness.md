## Known Limitations

These limitations are documented in `docs/releases/v0.1.0.md` and `models/latest/model_card.md`:

1. **Single-season dataset.** 380 matches from EPL 2023/24 only. CV accuracy (44.2% ± 4.7%) reflects the small training size.
2. **No SHAP explainability.** Per-prediction feature contributions are planned for Stage 8.
3. **ELO ratings reset per run.** No cross-season persistence; starts at 1500 on every pipeline execution.
4. **No backend or API.** Predictions are accessible via the inference module only; no HTTP API exists yet.
5. **No Android screens connected to data.** The Compose Multiplatform foundation is complete; screens are placeholders.
6. **No AI assistant.** RAG pipeline and Ollama integration are planned for Stage 10.
7. **Absolute Windows path in model registry.** The `run_dir` field stores the local absolute path of the machine that ran training.
8. **External data dependency.** Ingestion requires network access to football-data.co.uk.

## Release Recommendation

**Recommendation: APPROVED for public release as v0.1.0.**

This release represents a complete, tested, and documented implementation of Stages 1–7. The codebase demonstrates AI engineering discipline: grounded data pipelines, reproducible training, honest evaluation metrics, and a traceable model registry. All quality gates pass. All documented CLI commands work. All known limitations are documented honestly.

The repository is suitable for:
- Portfolio presentation and technical interviews.
- Demonstration of practical AI engineering practices.
- A starting point for contributors joining at Stage 8.

The repository is not suitable for:
- Production deployment (no backend, no authentication, no scaling).
- Gambling or commercial prediction use cases (explicitly prohibited in the model card).
