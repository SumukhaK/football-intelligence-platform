# Code of Conduct

## Our Standards

This project is a professional engineering environment. All contributors are expected to:

- Communicate respectfully and constructively.
- Provide feedback on code and ideas, not on people.
- Accept technical decisions that have been made through the documented architecture process.
- Raise disagreements through the ADR process, not through commit bypasses or undocumented changes.
- Keep discussions focused on the technical problem at hand.

## Unacceptable Behaviour

- Personal attacks or dismissive language toward contributors.
- Bypassing the architecture or review process without documented justification.
- Committing untested or undocumented changes to shared branches.
- Introducing dependencies, frameworks, or design changes without an ADR.

## Reporting

If you observe behaviour that violates these standards, raise it privately with the project maintainer.

## Scope

This code of conduct applies to all project spaces: the repository, pull requests, issues, and any communication about the project.
